The Range class Open on JetBrains Academy

The Range class is supposed to represent an integer range. Its invariants restrict the start and the end of the range so that the value of thefrom field is not greater than the one of the to field.

This class is serializable and there's a chance that the invariants can break during serialization/deserialization. Improve the class definition so that it preserves class invariants during deserialization.

If the deserialized object breaks the invariant, throw IllegalArgumentException like a constructor does.
